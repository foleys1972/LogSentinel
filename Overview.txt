LogSentinel Enterprise - Enhanced ML-Powered Log Monitoring System
🎯 Complete System Overview
LogSentinel Enhanced is a professional, real-time log monitoring system with integrated ML anomaly detection, interactive mapping, and enterprise-grade data processing capabilities.
Core Capabilities

Real-time Log Processing: Continuous tailing of log files across multiple site folders
ML Anomaly Detection: Live learning system that identifies suspicious patterns without training data
Interactive Mapping: Maps integration with RAG status indicators
Data Enrichment: Automatic correlation of IPs, MACs, and error codes to business context
Enterprise Import: CSV/Excel field mapping for error codes, assets, and system components
Real-time Dashboard: WebSocket-powered auto-refreshing interface with Chart.js visualizations


🏗️ Technical Architecture
Technology Stack

Backend: FastAPI + Python 3.9+
Database: PostgreSQL with time-series partitioning
ML Engine: Scikit-learn + Pandas + NumPy (lightweight approach)
Task Queue: Celery for ML training/prediction background processing
Real-time: WebSocket connections for live updates
Frontend: Vanilla JavaScript + Chart.js + Google Maps API
Design: Modern glassmorphism UI with responsive layout

System Architecture Flow
Log Files → File Watcher → Real-time Parser → Enrichment Engine → PostgreSQL
                                          ↓
WebSocket ← Dashboard ← ML Anomaly Engine ← Background ML Training (Celery)
    ↓
Google Maps + RAG Status + Chart.js Visualizations

🤖 ML Anomaly Detection Specifications
Detection Types Implemented

Pattern-based: Unusual log frequency spikes/drops using statistical thresholds
Behavior-based: Abnormal user access patterns via clustering analysis
Time-series: Seasonal/hourly pattern deviations using moving averages
Threshold-based: Error rate exceeding dynamic baselines
Clustering: New error types detection using DBSCAN

ML Implementation Details

Live Learning: No pre-trained models - system learns from incoming logs
Baseline Establishment: 4 weeks of data to establish "normal" patterns
Training Schedule: Models retrain monthly via Celery background tasks
Anomaly Scoring: Ranked suspicious events (0-100 scale)
Features Analyzed: Log frequency, error patterns, user behavior, IP access patterns, time-based trends

ML Algorithms Used

Isolation Forest: For outlier detection in log patterns
DBSCAN Clustering: For identifying new error types
Statistical Analysis: Z-score and moving averages for threshold detection
Time Series Analysis: Seasonal decomposition for pattern recognition


🗺️ Interactive Map Implementation
Google Maps Integration

Technology: Google Maps JavaScript API v3
Geocoding: Automatic address-to-coordinates conversion from location field
Map Markers: Custom RAG status indicators (Red/Amber/Green)
Interaction: Click markers to drill down to site-specific alerts and health details

RAG Status Calculation

Green: Site Health > 80 (healthy, minimal alerts)
Amber: Site Health 50-80 (warning, moderate alert activity)
Red: Site Health < 50 (critical, high alert volume or severity)

Site Health Algorithm
pythonSite Health = Base(100) - Critical(×10) - High(×5) - Medium(×2) - Low(×1)
# Updated every 5 minutes based on last 24 hours of alerts

⚡ Real-time Processing Specifications
Log Processing Pipeline

File Watcher: Monitors main folder + all child folders recursively
Real-time Tailing: Continuous log file monitoring using watchdog library
Parsing Engine: Immediate log line processing and enrichment
WebSocket Broadcasting: Live updates pushed to connected dashboards
ML Processing: Background anomaly analysis via Celery tasks

Performance Specifications

Log Volume: 10,000 lines per hour per site
Processing Latency: < 2 seconds from log entry to dashboard display
Retention: 6 months of detailed logs, 2 years of aggregated data
WebSocket Updates: 30-second refresh cycle with incremental updates
ML Analysis: Real-time scoring with monthly model retraining
